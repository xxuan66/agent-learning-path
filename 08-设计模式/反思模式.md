# 反思模式

> 自我评估和改进：让 Agent 能够反思自己的输出并持续改进。

---

## 核心思想

Agent 不仅执行任务，还能：
1. **评估输出**：检查自己的回答质量
2. **发现问题**：识别不足之处
3. **自我改进**：修正问题，提升质量

---

## 工作流程

```
用户任务
    │
    ▼
┌─────────┐
│  生成   │ ← 生成初始输出
└────┬────┘
     │
     ▼
┌─────────┐
│  评估   │ ← 检查输出质量
└────┬────┘
     │
     ▼
┌─────────┐
│  反思   │ ← 发现问题，提出改进
└────┬────┘
     │
     ▼
  是否需要改进？
     │
  ┌──┴──┐
  │否   │是
  ▼     ▼
输出   ┌─────────┐
       │  改进   │ ← 根据反思修改
       └────┬────┘
            │
            └──► 回到"评估"
```

---

## 三种角色

反思模式通常包含三个角色：

| 角色 | 职责 | 类比 |
|------|------|------|
| **生成者** | 生成初始输出 | 作者 |
| **评估者** | 评估输出质量 | 评审人 |
| **改进者** | 根据反馈改进 | 编辑 |

## 代码示例

```python
class ReflexionAgent:
    def __init__(self, llm, max_iterations=3):
        self.llm = llm
        self.max_iterations = max_iterations
    
    def run(self, task):
        # 1. 生成初始输出
        output = self.generate(task)
        
        for i in range(self.max_iterations):
            # 2. 评估输出
            evaluation = self.evaluate(task, output)
            
            # 3. 判断是否需要改进
            if evaluation["score"] >= 8:
                return output
            
            # 4. 反思和改进
            reflection = self.reflect(evaluation)
            output = self.improve(output, reflection)
        
        return output
    
    def generate(self, task):
        prompt = f"请完成以下任务：\n{task}"
        return self.llm.call(prompt)
    
    def evaluate(self, task, output):
        prompt = f"""请评估以下输出的质量：

任务：{task}
输出：{output}

评估标准：
1. 准确性（1-10分）
2. 完整性（1-10分）
3. 清晰度（1-10分）
4. 实用性（1-10分）

请用 JSON 格式输出：
{{
    "score": 总分,
    "accuracy": 分数,
    "completeness": 分数,
    "clarity": 分数,
    "usefulness": 分数,
    "issues": ["问题1", "问题2"],
    "suggestions": ["建议1", "建议2"]
}}"""
        return json.loads(self.llm.call(prompt))
    
    def reflect(self, evaluation):
        prompt = f"""基于以下评估结果，分析问题原因并提出改进方案：

评估结果：{evaluation}

请分析：
1. 问题的根本原因
2. 改进的具体方向
3. 具体的改进措施"""
        return self.llm.call(prompt)
    
    def improve(self, output, reflection):
        prompt = f"""请根据以下反思改进输出：

原输出：{output}
反思：{reflection}

请生成改进后的输出。"""
        return self.llm.call(prompt)
```

### 改进后
```python
def binary_search(arr: list[int], target: int) -> int:
    """
    二分查找算法实现
    
    Args:
        arr: 已排序的整数数组
        target: 要查找的目标值
    
    Returns:
        目标值的索引，未找到返回 -1
    
    Raises:
        ValueError: 数组为空时抛出
    
    Example:
        >>> binary_search([1, 2, 3, 4, 5], 3)
        2
        >>> binary_search([1, 2, 3, 4, 5], 6)
        -1
    """
    if not arr:
        raise ValueError("数组不能为空")
    
    left, right = 0, len(arr) - 1
    
    while left <= right:
        mid = (left + right) // 2
        if arr[mid] == target:
            return mid
        elif arr[mid] < target:
            left = mid + 1
        else:
            right = mid - 1
    
    return -1
```

## 优缺点

### 优点

- **质量更高**：通过迭代提升质量
- **自我纠错**：自动发现和修复问题
- **可解释**：反思过程可见
- **适用性广**：各种任务都能用

### 缺点

- **成本更高**：多次 LLM 调用
- **速度较慢**：需要多次迭代
- **可能过度优化**：在简单任务上浪费资源

---

## 适用场景

- 代码生成和审查
- 文章写作和润色
- 翻译质量提升
- 关键决策

## 不适用场景

- 简单查询
- 时间敏感任务
- 成本敏感场景

## 改进方向

### 1. 外部评估

```python
# 使用外部工具评估
def evaluate_with_tests(code):
    result = run_tests(code)
    return {
        "passed": result.passed,
        "failed": result.failed,
        "coverage": result.coverage
    }
```

### 2. 多角度反思

```python
# 从不同角度反思
reflections = [
    self.reflect_as_expert(evaluation),    # 专家视角
    self.reflect_as_beginner(evaluation),  # 新手视角
    self.reflect_as_critic(evaluation)     # 批评者视角
]
```

### 3. 反思记忆

```python
# 记录反思历史，避免重复错误
self.memory.store({
    "task": task,
    "issues": evaluation["issues"],
    "solutions": reflection
})
```
