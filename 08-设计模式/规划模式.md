# 规划模式

> 先规划再执行：将复杂任务拆分为可管理的子任务。

---

## 核心思想

复杂任务难以一步完成，需要：
1. **理解任务**：分析任务要求
2. **制定计划**：拆分为子任务
3. **执行计划**：按顺序完成子任务
4. **整合结果**：汇总输出

---


## 规划器实现

规划器的核心工作是：接收一个复杂任务，让 LLM 把它拆成若干子任务，并指定每个子任务的依赖关系。输出是 JSON 格式，方便后续程序化处理。

```python
class Planner:
    def __init__(self, llm):
        self.llm = llm
    
    def create_plan(self, task):
        prompt = f"""请将以下任务拆分为可执行的子任务：

任务：{task}

要求：
1. 每个子任务应该是独立的、可执行的
2. 子任务之间有明确的依赖关系
3. 每个子任务有清晰的输入输出

请用 JSON 格式输出：
{{
    "subtasks": [
        {{
            "id": 1,
            "name": "子任务名称",
            "description": "子任务描述",
            "dependencies": [],
            "input": "输入说明",
            "output": "输出说明"
        }}
    ]
}}"""
        
        response = self.llm.call(prompt)
        return self.parse_plan(response)
    
    def parse_plan(self, response):
        # 解析 JSON 格式的计划
        import json
        return json.loads(response)
```

## 执行器实现

执行器负责按依赖顺序执行子任务。核心逻辑是一个 while 循环：每轮找出所有"依赖已完成"的子任务，执行它们，直到全部完成。如果某一轮找不到可执行的任务，说明存在循环依赖。

```python
class PlanExecutor:
    def __init__(self, agent, tools):
        self.agent = agent
        self.tools = tools
        self.results = {}
    
    def execute(self, plan):
        # 按依赖顺序执行
        completed = set()
        
        while len(completed) < len(plan["subtasks"]):
            # 找到可执行的任务（依赖已完成）
            ready = [
                task for task in plan["subtasks"]
                if task["id"] not in completed
                and all(dep in completed for dep in task["dependencies"])
            ]
            
            if not ready:
                raise Exception("存在循环依赖")
            
            # 执行任务
            for task in ready:
                result = self.execute_task(task)
                self.results[task["id"]] = result
                completed.add(task["id"])
        
        return self.results
    
    def execute_task(self, task):
        # 获取依赖的输入
        inputs = {
            dep: self.results[dep]
            for dep in task["dependencies"]
        }
        
        # 执行任务
        prompt = f"""执行任务：{task['name']}
描述：{task['description']}
输入：{inputs if inputs else task['input']}

请完成任务并返回结果。"""
        
        return self.agent.run(prompt)
```

## 完整示例

把规划器和执行器组合在一起，就是完整的规划模式 Agent：先规划、再执行、最后整合结果。

```python
class PlanningAgent:
    def __init__(self, llm, tools):
        self.planner = Planner(llm)
        self.executor = PlanExecutor(Agent(llm, tools), tools)
    
    def run(self, task):
        # 1. 规划
        print("📋 正在制定计划...")
        plan = self.planner.create_plan(task)
        self.print_plan(plan)
        
        # 2. 执行
        print("\n🚀 开始执行计划...")
        results = self.executor.execute(plan)
        
        # 3. 整合
        print("\n📝 整合结果...")
        final = self.integrate_results(task, results)
        
        return final
    
    def print_plan(self, plan):
        for task in plan["subtasks"]:
            deps = f" (依赖: {task['dependencies']})" if task['dependencies'] else ""
            print(f"  {task['id']}. {task['name']}{deps}")
    
    def integrate_results(self, task, results):
        prompt = f"""任务：{task}

各子任务结果：
{results}

请整合以上结果，生成最终输出。"""
        return self.llm.call(prompt)
```

## 优缺点

### 优点

- **结构清晰**：任务分解明确
- **可控性强**：可以审查和调整计划
- **可追溯**：每个步骤有记录
- **适合复杂任务**：处理多步骤任务

### 缺点

- **规划成本**：需要额外的 LLM 调用
- **不够灵活**：计划可能需要调整
- **依赖规划质量**：规划不好则执行差
