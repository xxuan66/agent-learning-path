# Agent 设计模式

> 借鉴软件工程的设计模式思想，总结 Agent 开发中的常见模式。

---

## 什么是设计模式

设计模式是解决特定问题的可复用方案。在 Agent 开发中，我们同样面临一些反复出现的问题，可以总结为设计模式。

---

## 已整理的模式

- [ReAct 模式](ReAct%20模式.md) - 边推理边行动，最基础也最常用
- [规划模式](规划模式.md) - 先规划再执行，处理复杂任务
- [反思模式](反思模式.md) - 自我评估和改进，持续提升质量

---

## 模式速查表

| 模式 | 解决问题 | 适用场景 | 复杂度 |
|------|---------|---------|--------|
| ReAct | 需要边想边做 | 通用任务 | ⭐⭐ |
| 规划 | 任务复杂需拆分 | 多步骤任务 | ⭐⭐⭐ |
| 反思 | 需要自我改进 | 质量要求高 | ⭐⭐⭐ |

## 组合使用

实际项目中，通常组合多种模式：

```
用户请求
    │
    ▼
┌─────────┐
│ 规划模式 │ ← 先拆分任务
└────┬────┘
     │
     ▼
┌─────────┐
│ 链式模式 │ ← 按顺序执行
└────┬────┘
     │
     ▼
┌─────────┐
│ 反思模式 │ ← 质量检查
└────┬────┘
     │
     ▼
  最终输出
```

---

## 最佳实践

1. **从简单开始**：先用 ReAct 模式，不够再加
2. **按需组合**：不要过度设计
3. **保持一致**：同一个系统内尽量统一模式
4. **文档化**：记录选择的模式和原因
