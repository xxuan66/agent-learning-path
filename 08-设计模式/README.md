# Agent 设计模式

> 借鉴软件工程的设计模式思想，总结 Agent 开发中的常见模式。

---

## 什么是设计模式

设计模式是解决特定问题的可复用方案。在 Agent 开发中，我们同样面临一些反复出现的问题，可以总结为设计模式。

---

## 模式分类

### 创建型模式
- [工厂模式](工厂模式.md) - 灵活创建不同类型的 Agent
- [配置驱动模式](配置驱动模式.md) - 通过配置定义 Agent 行为

### 结构型模式
- [代理模式](代理模式.md) - Agent 作为中介访问资源
- [装饰器模式](装饰器模式.md) - 增强 Agent 能力
- [组合模式](组合模式.md) - 构建 Agent 层次结构

### 行为型模式
- [ReAct 模式](ReAct模式.md) - 推理与行动交替
- [规划模式](规划模式.md) - 先规划后执行
- [反思模式](反思模式.md) - 自我评估和改进
- [投票模式](投票模式.md) - 多 Agent 投票决策
- [链式模式](链式模式.md) - 任务串联执行

---

## 模式速查表

| 模式 | 解决问题 | 适用场景 | 复杂度 |
|------|---------|---------|--------|
| ReAct | 需要边想边做 | 通用任务 | ⭐⭐ |
| 规划 | 任务复杂需拆分 | 多步骤任务 | ⭐⭐⭐ |
| 反思 | 需要自我改进 | 质量要求高 | ⭐⭐⭐ |
| 投票 | 需要集体决策 | 关键决策 | ⭐⭐⭐ |
| 链式 | 任务有明确顺序 | 流程化任务 | ⭐⭐ |
| 工厂 | 需要多种 Agent | 复杂系统 | ⭐⭐ |
| 代理 | 需要访问控制 | 安全场景 | ⭐⭐ |
| 装饰器 | 需要增强功能 | 功能扩展 | ⭐⭐ |
| 组合 | 需要层次结构 | 复杂组织 | ⭐⭐⭐⭐ |

## 组合使用

实际项目中，通常组合多种模式：

```
用户请求
    │
    ▼
┌─────────┐
│ 规划模式 │ ← 先拆分任务
└────┬────┘
     │
     ▼
┌─────────┐
│ 链式模式 │ ← 按顺序执行
└────┬────┘
     │
     ▼
┌─────────┐
│ 反思模式 │ ← 质量检查
└────┬────┘
     │
     ▼
  最终输出
```

## 最佳实践

1. **从简单开始**：先用 ReAct 模式，不够再加
2. **按需组合**：不要过度设计
3. **保持一致**：同一个系统内尽量统一模式
4. **文档化**：记录选择的模式和原因
