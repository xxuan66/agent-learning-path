# 部署指南

## 🚀 推送到 GitHub

### 方法一：使用 GitHub CLI

```bash
# 1. 安装 gh CLI（如果未安装）
# macOS: brew install gh
# Linux: sudo apt install gh

# 2. 登录 GitHub
gh auth login

# 3. 创建仓库
gh repo create agent-learning-guide --public --source=. --push

# 完成！访问 https://github.com/你的用户名/agent-learning-guide
```

### 方法二：手动创建

```bash
# 1. 在 GitHub 创建新仓库
# 访问 https://github.com/new
# 仓库名：agent-learning-guide
# 可见性：Public
# 不要初始化 README（我们已经有了）

# 2. 添加远程仓库
git remote add origin https://github.com/你的用户名/agent-learning-guide.git

# 3. 推送代码
git branch -M main
git push -u origin main
```

### 方法三：使用 Git 凭证

```bash
# 配置凭证
git config --global credential.helper store

# 添加远程
git remote add origin https://github.com/你的用户名/agent-learning-guide.git

# 推送（会提示输入用户名和密码/Token）
git push -u origin main
```

## 📝 使用 GitHub Token

```bash
# 1. 创建 Token
# 访问 https://github.com/settings/tokens
# 选择 repo 权限
# 复制 Token

# 2. 使用 Token 推送
git remote add origin https://你的用户名:TOKEN@github.com/你的用户名/agent-learning-guide.git
git push -u origin main
```

### 添加贡献指南

创建 `CONTRIBUTING.md`：

```markdown
# 贡献指南

欢迎贡献！请参考以下步骤：

1. Fork 本仓库
2. 创建特性分支 (`git checkout -b feature/AmazingFeature`)
3. 提交更改 (`git commit -m 'Add some AmazingFeature'`)
4. 推送到分支 (`git push origin feature/AmazingFeature`)
5. 开启 Pull Request
```

```markdown

1. ...
2. ...

**期望行为**
应该发生什么

**实际行为**
实际发生了什么
```

## 📊 GitHub Pages（可选）

如果想生成在线文档：

```bash
# 1. 安装 mkdocs
pip install mkdocs mkdocs-material

# 2. 初始化
mkdocs new docs-site
cd docs-site

# 3. 配置 mkdocs.yml
# 4. 构建
mkdocs build

# 5. 推送到 gh-pages 分支
ghp-import -n -p -f docs-site/build
```

## 🔔 启用 GitHub Actions

创建 `.github/workflows/ci.yml` 实现自动测试和部署。
