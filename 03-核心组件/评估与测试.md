# Agent 评估与测试

> 如何衡量 Agent 的能力？如何确保 Agent 在生产环境中可靠？

---

## 为什么需要评估

- **验证能力**：Agent 是否能完成预期任务
- **比较方案**：不同设计、不同模型的效果对比
- **发现问题**：定位 Agent 的薄弱环节
- **持续改进**：建立基线，量化改进效果

---

## 评估维度

### 1. 任务完成度

Agent 是否成功完成了用户要求的任务？

| 等级 | 描述 |
|------|------|
| ✅ 完全完成 | 任务目标完全达成 |
| ⚠️ 部分完成 | 完成了主要目标，但有遗漏 |
| ❌ 未完成 | 任务失败或结果错误 |

### 2. 效率

Agent 完成任务消耗了多少资源？

```python
efficiency_metrics = {
    "llm_calls": 5,           # LLM 调用次数
    "tokens_used": 2500,      # 消耗的 token 数
    "tool_calls": 3,          # 工具调用次数
    "time_seconds": 12.5,     # 总耗时
    "cost_usd": 0.05          # 成本（美元）
}
```

### 3. 准确性

Agent 的输出是否正确？

- **事实准确性**：信息是否真实
- **逻辑准确性**：推理过程是否合理
- **格式准确性**：输出格式是否符合要求

### 4. 鲁棒性

Agent 在异常情况下表现如何？

- 输入格式变化
- 工具返回异常
- 网络超时
- 边界情况

### 5. 安全性

Agent 是否会产生有害输出？

- 是否泄露系统提示词
- 是否执行危险操作
- 是否产生不当内容

## 评估方法

### 1. 人工评估

**优点**：最准确，能评估主观质量

**缺点**：成本高，不可扩展

**适用场景**：小规模验证，关键任务

```
评估流程：
1. 准备测试用例（20-50个）
2. 运行 Agent 获取输出
3. 评估者打分（1-5分）
4. 计算平均分和分布
```

### 2. 自动评估

**优点**：成本低，可扩展，可重复

**缺点**：可能不够准确

**适用场景**：大规模测试，回归测试

```python
# 使用 LLM 评估 LLM 输出
from langchain_openai import ChatOpenAI

llm = ChatOpenAI(model="gpt-4", temperature=0)

def evaluate_with_llm(input, output, criteria):
    prompt = f"""
    请评估以下 Agent 输出的质量：
    
    用户输入：{input}
    Agent 输出：{output}
    
    评估标准：
    {criteria}
    
    请给出 1-5 分的评分和理由。
    """
    return llm.invoke(prompt).content
```

### 3. 基准测试

使用标准化的数据集和指标。

**常用基准**：

| 基准 | 评估内容 | 数据规模 |
|------|---------|---------|
| **AgentBench** | 综合能力 | 8个环境 |
| **WebArena** | 网页操作 | 812个任务 |
| **SWE-bench** | 代码修复 | 2294个问题 |
| **ToolBench** | 工具使用 | 16000+ API |
| **API-Bank** | API 调用 | 73个 API |

## 测试策略

### 1. 单元测试

测试 Agent 的单个组件。

```python
# 测试工具调用
def test_search_tool():
    result = search_tool("Python教程")
    assert "Python" in result
    assert len(result) > 0

# 测试记忆系统
def test_memory_storage():
    memory.store("key", "value")
    assert memory.retrieve("key") == "value"
```

### 2. 集成测试

测试 Agent 的完整流程。

```python
from langchain_openai import ChatOpenAI
from langchain.agents import AgentExecutor, create_tool_calling_agent
from langchain_core.prompts import ChatPromptTemplate

# 测试完整的 Agent 交互
def test_agent_workflow():
    llm = ChatOpenAI(model="gpt-4", temperature=0)
    prompt = ChatPromptTemplate.from_messages([
        ("system", "你是一个有帮助的助手"),
        ("human", "{input}"),
    ])
    agent = create_tool_calling_agent(llm, [], prompt)
    agent_executor = AgentExecutor(agent=agent, tools=[], verbose=True)
    
    response = agent_executor.invoke({"input": "查询明天的天气"})
    
    assert "output" in response
    assert "天气" in response["output"]
```

### 3. 端到端测试

模拟真实用户场景。

```python
# 测试真实场景
def test_real_scenario(agent_executor):
    scenarios = [
        {
            "input": "帮我订一张明天去北京的机票",
            "expected_actions": ["search_flights", "book_flight"],
            "success_criteria": lambda r: "已预订" in r["output"]
        }
    ]
    
    for scenario in scenarios:
        result = agent_executor.invoke({"input": scenario["input"]})
        assert scenario["success_criteria"](result)
```

### 4. 压力测试

测试 Agent 在极端情况下的表现。

```python
# 并发测试
async def test_concurrent_requests(agent_executor):
    tasks = [agent_executor.ainvoke({"input": f"任务{i}"}) for i in range(100)]
    results = await asyncio.gather(*tasks)
    assert all("output" in r for r in results)

# 长对话测试
def test_long_conversation(agent_executor):
    for i in range(50):
        response = agent_executor.invoke({"input": f"第{i}个问题"})
        assert response is not None
```

## 评估框架示例

```python
class AgentEvaluator:
    def __init__(self, agent, test_cases):
        self.agent = agent
        self.test_cases = test_cases
        self.results = []
    
    def run_evaluation(self):
        for case in self.test_cases:
            result = self.agent.run(case["input"])
            score = self.evaluate_single(result, case)
            self.results.append({
                "input": case["input"],
                "output": result,
                "expected": case["expected"],
                "score": score
            })
        return self.generate_report()
    
    def evaluate_single(self, result, case):
        score = 0
        if result.status == "success":
            score += 40
        if case["expected"] in result.content:
            score += 40
        if result.tool_calls <= case.get("max_tools", 5):
            score += 20
        return score
    
    def generate_report(self):
        avg_score = sum(r["score"] for r in self.results) / len(self.results)
        return {
            "total_cases": len(self.results),
            "average_score": avg_score,
            "pass_rate": sum(1 for r in self.results if r["score"] >= 60) / len(self.results),
            "details": self.results
        }
```

## 持续评估

### CI/CD 集成

```yaml
# .github/workflows/agent-test.yml
name: Agent Tests
on: [push, pull_request]
jobs:
  test:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v2
      - name: Run Agent Tests
        run: python -m pytest tests/agent/
      - name: Run Evaluation
        run: python evaluation/run_benchmark.py
```

### 回归测试

每次更新后运行测试，确保没有破坏现有功能。

```bash
# 运行完整测试套件
python -m pytest tests/ -v

# 只运行失败的测试
python -m pytest tests/ --lf

# 运行特定测试
python -m pytest tests/test_agent.py::test_search
```

## 最佳实践

1. **建立测试用例库**：收集真实用户请求作为测试用例
2. **多维度评估**：不要只看准确率，也要看效率和成本
3. **自动化优先**：能自动化的测试就不要手动
4. **定期评估**：模型更新后重新评估
5. **记录历史**：追踪 Agent 能力的变化趋势
