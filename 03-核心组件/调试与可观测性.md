# Agent 调试与可观测性

> 当 Agent 行为异常时，如何快速定位问题？

---

## 为什么需要可观测性

Agent 是"黑盒"系统：
- 决策过程不透明
- 多次 LLM 调用难以追踪
- 工具调用链复杂
- 错误可能在任何环节发生

可观测性帮助我们：
- 理解 Agent 在做什么
- 定位问题发生的位置
- 优化性能和成本
- 建立对 Agent 的信任

---

## 可观测性三大支柱

### 1. 日志（Logging）

日志是可观测性的基础——记录 Agent 每一步在做什么。下面的代码定义了一个简单的日志函数，每次 Agent 执行一个步骤时，记录"做了什么、输入了什么、输出了什么、消耗了多少 token"。

```python
import logging

logger = logging.getLogger("agent")

def agent_step(step_name, input_data, output_data):
    logger.info(f"Step: {step_name}")
    logger.info(f"Input: {input_data}")
    logger.info(f"Output: {output_data}")
    logger.info(f"Tokens: {output_data.get('tokens', 'N/A')}")
```

**日志级别**：

| 级别 | 用途 |
|------|------|
| DEBUG | 详细调试信息 |
| INFO | 正常操作记录 |
| WARNING | 潜在问题 |
| ERROR | 错误但可恢复 |
| CRITICAL | 严重错误 |

### 2. 指标（Metrics）

指标把 Agent 的运行状态量化成数字，方便监控和告警。下面列举了 Agent 系统最需要关注的四类指标：

```python
# 关键指标
metrics = {
    # 性能指标
    "latency_ms": 1250,           # 响应延迟
    "tokens_per_second": 45,      # 生成速度
    
    # 成本指标
    "tokens_used": 2500,          # 消耗 token
    "cost_usd": 0.05,             # 费用
    
    # 质量指标
    "success_rate": 0.95,         # 成功率
    "tool_call_count": 3,         # 工具调用次数
    
    # 资源指标
    "memory_mb": 256,             # 内存使用
    "cpu_percent": 15             # CPU 使用
}
```

### 3. 追踪（Tracing）

追踪记录一次请求的完整生命周期。下面的代码定义了 Trace（追踪）和 Span（跨度）的概念：

- **Trace**：一次完整请求的追踪记录，比如用户问了一个问题到拿到回答的全过程
- **Span**：Trace 中的一个步骤，比如一次 LLM 调用或一次工具调用
- **parent_id**：表示 Span 之间的父子关系，比如一个 LLM 调用 Span 下面挂了一个工具调用 Span

```python
class Trace:
    def __init__(self, trace_id):
        self.trace_id = trace_id
        self.spans = []
    
    def start_span(self, name, parent_id=None):
        span = {
            "span_id": generate_id(),
            "parent_id": parent_id,
            "name": name,
            "start_time": time.time()
        }
        self.spans.append(span)
        return span["span_id"]
    
    def end_span(self, span_id, output=None):
        for span in self.spans:
            if span["span_id"] == span_id:
                span["end_time"] = time.time()
                span["output"] = output
                break
```

## 调试工具

### 1. LangSmith

LangSmith 是 LangChain 官方的可观测性平台。下面的代码展示了如何启用追踪：关键是在创建 `AgentExecutor` 时传入 `callbacks=[tracer]`，之后所有的 LLM 调用和工具调用都会被自动记录，可以在 LangSmith 的网页界面上查看完整的调用链。

```python
from langsmith import Client
from langchain_core.tracers import LangChainTracer
from langchain_openai import ChatOpenAI
from langchain.agents import AgentExecutor, create_tool_calling_agent
from langchain_core.prompts import ChatPromptTemplate

# 启用追踪
tracer = LangChainTracer()
llm = ChatOpenAI(model="gpt-4", temperature=0)

# 创建带追踪的 Agent
prompt = ChatPromptTemplate.from_messages([
    ("system", "你是一个有帮助的助手"),
    ("human", "{input}"),
])
agent = create_tool_calling_agent(llm, [], prompt)
agent_executor = AgentExecutor(
    agent=agent,
    tools=[],
    callbacks=[tracer],  # 启用追踪
    verbose=True
)

# 自动记录所有调用
result = agent_executor.invoke({"input": "查询天气"})
```

**功能**：
- 自动追踪所有 LLM 和工具调用
- 可视化调用链
- 评估和测试集成
- 数据集管理

### 2. Langfuse

开源的 LLM 可观测性平台，可以自托管。用法和 LangSmith 类似——通过 callback 机制自动追踪 Agent 的所有操作。

```python
from langfuse.callback import CallbackHandler
from langchain_openai import ChatOpenAI
from langchain.agents import AgentExecutor, create_tool_calling_agent
from langchain_core.prompts import ChatPromptTemplate

handler = CallbackHandler()
llm = ChatOpenAI(model="gpt-4", temperature=0)

# 创建带追踪的 Agent
prompt = ChatPromptTemplate.from_messages([
    ("system", "你是一个有帮助的助手"),
    ("human", "{input}"),
])
agent = create_tool_calling_agent(llm, [], prompt)
agent_executor = AgentExecutor(
    agent=agent,
    tools=[],
    callbacks=[handler],
    verbose=True
)

result = agent_executor.invoke({"input": "分析数据"})
```

**功能**：
- 成本追踪
- 延迟分析
- 用户反馈收集
- 自托管支持

### 3. 自定义调试器

如果你不想引入第三方平台，也可以自己写一个简单的调试器。下面的代码实现了一个最小可用的调试工具：记录每一步操作和每次 LLM 调用的 token 消耗，最后输出汇总。

```python
class AgentDebugger:
    def __init__(self):
        self.logs = []
        self.traces = []
    
    def log_step(self, step_type, data):
        entry = {
            "timestamp": datetime.now(),
            "type": step_type,
            "data": data
        }
        self.logs.append(entry)
        print(f"[{step_type}] {data}")
    
    def trace_llm_call(self, prompt, response, tokens):
        trace = {
            "prompt": prompt,
            "response": response,
            "tokens": tokens,
            "timestamp": datetime.now()
        }
        self.traces.append(trace)
    
    def print_summary(self):
        print(f"Total steps: {len(self.logs)}")
        print(f"Total LLM calls: {len(self.traces)}")
        print(f"Total tokens: {sum(t['tokens'] for t in self.traces)}")
```

## 常见问题诊断

### 问题 1：Agent 陷入循环

**症状**：Agent 重复执行相同的操作。

**诊断方法**：
```python
def detect_loop(actions, threshold=3):
    """检测是否陷入循环"""
    recent_actions = actions[-threshold:]
    if len(set(str(a) for a in recent_actions)) == 1:
        return True
    return False
```

**解决方案**：
- 设置最大步数限制
- 添加动作去重
- 引入随机性打破循环

### 问题 2：工具调用失败

**症状**：Agent 调用工具后报错。

**诊断方法**：
```python
def diagnose_tool_error(error):
    error_types = {
        "APIError": "API 调用失败，检查 key 和额度",
        "TimeoutError": "超时，检查网络或增加超时时间",
        "ValidationError": "参数错误，检查工具定义",
        "RateLimitError": "触发限流，添加重试机制"
    }
    return error_types.get(type(error).__name__, "未知错误")
```

### 问题 3：输出质量差

**症状**：Agent 输出不符合预期。

**诊断方法**：
```python
def diagnose_quality_issue(prompt, output, expected):
    issues = []
    
    if len(output) < 50:
        issues.append("输出过短，可能被截断")
    
    if "我不确定" in output or "抱歉" in output:
        issues.append("Agent 表达不确定性")
    
    if expected not in output:
        issues.append(f"缺少预期内容: {expected}")
    
    return issues
```

## 最佳实践

### 1. 分级日志

```python
# 开发环境：详细日志
if ENV == "development":
    logging.basicConfig(level=logging.DEBUG)

# 生产环境：只记录关键信息
if ENV == "production":
    logging.basicConfig(level=logging.WARNING)
```

### 2. 结构化日志

```python
# 使用 JSON 格式，便于分析
logger.info(json.dumps({
    "event": "llm_call",
    "model": "gpt-4",
    "tokens": 1500,
    "latency_ms": 1250
}))
```

### 3. 关联追踪

```python
# 使用 trace_id 关联所有相关日志
trace_id = generate_trace_id()
logger.info(f"[{trace_id}] Starting agent execution")
logger.info(f"[{trace_id}] LLM call completed")
logger.info(f"[{trace_id}] Tool call completed")
```

### 4. 设置告警

```python
def check_and_alert(metrics):
    if metrics["latency_ms"] > 10000:
        alert("Agent 响应过慢")
    
    if metrics["cost_usd"] > 1.0:
        alert("单次请求成本过高")
    
    if metrics["success_rate"] < 0.8:
        alert("成功率低于阈值")
```
