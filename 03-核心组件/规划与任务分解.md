# 规划与任务分解

规划是 Agent 从“会回答”走向“会执行”的关键一步。它决定系统面对复杂目标时，能不能把一个模糊任务拆成一串更清晰、更可完成的动作。

## 🎯 为什么需要规划？

大语言模型擅长：
- ✅ 理解自然语言
- ✅ 生成连贯文本
- ✅ 知识问答

但直接面对复杂任务时会：
- ❌ 缺乏系统性
- ❌ 容易遗漏步骤
- ❌ 难以处理长程依赖

**规划的作用**：将模糊的目标转化为清晰的执行路径。

但也要注意，不是所有 Agent 都需要复杂规划。很多简单场景里，明确的工具调用或固定工作流就已经足够。

## 📋 任务分解方法

### 1. 简单提示法

最直接的方法，通过提示引导 LLM 分解任务：

```python
from langchain.prompts import PromptTemplate

decompose_prompt = PromptTemplate.from_template("""
请将以下任务分解为可执行的步骤：

任务：{task}

要求：
1. 每个步骤应该是具体可执行的
2. 步骤之间应该有逻辑顺序
3. 标注步骤之间的依赖关系

分解结果：
""")

# 使用
chain = decompose_prompt | llm
result = chain.invoke({"task": "帮我策划一次公司团建活动"})
```

**示例输出**：
```
1. 确定团建预算和参与人数
2. 收集团建时间和地点偏好
3. 调研可选的团建活动方案
4. 比较各方案的成本和效果
5. 确定最终方案并预订
6. 通知参与人员并收集确认
7. 准备团建所需物资
8. 执行团建活动
9. 收集反馈并总结
```

### 2. Chain of Thought (CoT)

**核心思想**：让模型"逐步思考"

```python
cot_prompt = PromptTemplate.from_template("""
请逐步思考以下问题：

问题：{question}

让我们一步一步思考：

步骤 1: 首先，我需要理解问题的核心是...
步骤 2: 然后，我应该考虑...
步骤 3: 接下来，我可以...
步骤 4: 最后，得出结论...

答案：
""")
```

**效果对比**：

| 方法 | 准确率 | 适用场景 |
|------|--------|----------|
| 直接回答 | ~50% | 简单问题 |
| CoT | ~70% | 中等复杂度 |
| CoT + 示例 | ~80% | 复杂推理 |

### 3. Tree of Thoughts (ToT)

**核心思想**：探索多种可能性，选择最优路径

```python
class TreeOfThoughts:
    def __init__(self, llm, breadth=3, depth=4):
        self.llm = llm
        self.breadth = breadth  # 每层分支数
        self.depth = depth      # 最大深度
    
    def solve(self, problem):
        # 初始化
        thoughts = [problem]
        
        for level in range(self.depth):
            new_thoughts = []
            
            for thought in thoughts:
                # 生成多个可能的下一步
                candidates = self.generate_candidates(thought)
                
                # 评估每个候选
                scored = self.evaluate_candidates(candidates)
                
                # 选择最好的几个
                new_thoughts.extend(scored[:self.breadth])
            
            thoughts = new_thoughts
        
        # 返回最佳路径
        return self.select_best(thoughts)
```

**适用场景**：
- 创意写作
- 策略规划
- 复杂决策

### 4. LLM+P（结合经典规划器）

对于需要严格规划的场景，可以结合传统规划器：

```
LLM → PDDL 问题描述 → 经典规划器 → PDDL 计划 → LLM → 自然语言计划
```

## 一个更实用的判断

如果任务具有下面这些特点，规划通常会明显变得重要：

1. 任务是多步的
2. 中间步骤存在依赖关系
3. 执行中可能失败并需要调整
4. 最终结果不能只靠一次生成得到

## 📊 规划质量评估

| 指标 | 描述 | 优化方法 |
|------|------|----------|
| 完整性 | 是否覆盖所有必要步骤 | 增加验证步骤 |
| 可执行性 | 每个步骤是否具体可行 | 细化模糊步骤 |
| 顺序合理 | 步骤顺序是否符合逻辑 | 依赖关系检查 |
| 效率 | 是否有冗余步骤 | 优化和合并 |

## ⚠️ 常见陷阱

### 1. 过度分解

```
❌ 坏例子：
1. 打开电脑
2. 打开浏览器
3. 输入网址
4. 按回车
5. 等待页面加载
...

✅ 好例子：
1. 访问目标网站
2. 查找所需信息
```

### 2. 忽略依赖

```
❌ 坏例子：
1. 编写代码
2. 设计数据库
3. 测试功能

✅ 好例子：
1. 设计数据库 schema（前置）
2. 编写数据访问层代码（依赖 1）
3. 编写业务逻辑代码（依赖 2）
4. 集成测试（依赖 3）
```

### 3. 缺乏验证

## 🚀 进阶技巧

### 1. 分层规划

```
高层计划（战略层）
    ↓
中层计划（战术层）
    ↓
底层计划（执行层）
```

### 2. 动态调整

```python
class AdaptivePlanner:
    def execute_and_adjust(self, plan):
        for i, step in enumerate(plan):
            result = self.execute(step)
            
            if result.success:
                continue
            else:
                # 重新规划剩余步骤
                new_plan = self.replan(
                    remaining_steps=plan[i+1:],
                    context=result.context
                )
                plan = new_plan
```

### 3. 并行执行

识别可并行执行的步骤：

```python
def identify_parallel_steps(steps):
    """识别可以并行执行的步骤"""
    parallel_groups = []
    current_group = []
    
    for step in steps:
        if has_dependencies(step, current_group):
            parallel_groups.append(current_group)
            current_group = [step]
        else:
            current_group.append(step)
    
    parallel_groups.append(current_group)
    return parallel_groups
```

## 📚 代码示例：完整规划系统

```python
from pydantic import BaseModel
from typing import List, Optional

class Step(BaseModel):
    id: int
    description: str
    dependencies: List[int] = []
    estimated_time: str = ""
    status: str = "pending"

class Plan(BaseModel):
    objective: str
    steps: List[Step]
    
    def get_next_steps(self) -> List[Step]:
        """获取可执行的下一步"""
        completed = {s.id for s in self.steps if s.status == "completed"}
        
        next_steps = []
        for step in self.steps:
            if step.status == "pending":
                if all(dep in completed for dep in step.dependencies):
                    next_steps.append(step)
        
        return next_steps
    
    def mark_complete(self, step_id: int):
        """标记步骤完成"""
        for step in self.steps:
            if step.id == step_id:
                step.status = "completed"

class PlanningAgent:
    def __init__(self, llm):
        self.llm = llm
    
    def create_plan(self, objective: str) -> Plan:
        """创建计划"""
        prompt = f"""
        为以下目标创建详细计划：
        
        目标：{objective}
        
        输出格式：JSON，包含步骤列表，每个步骤有：
        - id: 步骤编号
        - description: 步骤描述
        - dependencies: 依赖的步骤 ID 列表
        - estimated_time: 预估时间
        """
        
        response = self.llm.generate(prompt)
        return Plan.parse(response)
    
    def execute_plan(self, plan: Plan):
        """执行计划"""
        while True:
            next_steps = plan.get_next_steps()
            
            if not next_steps:
                # 检查是否全部完成
                if all(s.status == "completed" for s in plan.steps):
                    print("计划执行完成！")
                    break
                else:
                    print("计划无法继续，存在阻塞")
                    break
            
            # 执行可并行的步骤
            for step in next_steps:
                print(f"执行步骤 {step.id}: {step.description}")
                result = self.execute_step(step)
                
                if result.success:
                    plan.mark_complete(step.id)
                else:
                    # 处理失败
                    self.handle_failure(step, result)
```

## 🎯 最佳实践

1. **明确目标**：确保规划前充分理解目标
2. **适度分解**：步骤不宜过粗或过细
3. **标注依赖**：清晰标注步骤间关系
4. **预留缓冲**：考虑意外情况
5. **持续验证**：执行过程中验证计划有效性
