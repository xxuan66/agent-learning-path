# 性能优化

性能优化在 Agent 系统里不只是“跑快一点”，而是直接影响成本、稳定性和用户体验。

如果一个 Agent 能回答，但太慢、太贵、太不稳定，它在真实场景里通常就不可用。

## ⚡ 优化策略

### 1. 缓存机制

```python
from functools import lru_cache
import hashlib

@lru_cache(maxsize=1000)
def cached_llm_call(prompt_hash: str, prompt: str) -> str:
    return llm.invoke(prompt)

def get_cached_response(prompt: str) -> str:
    prompt_hash = hashlib.md5(prompt.encode()).hexdigest()
    return cached_llm_call(prompt_hash, prompt)
```

### 2. 流式输出

```python
# 减少用户等待感知
async def stream_response(query: str):
    async for chunk in llm.astream(query):
        yield chunk.content

# 前端可以实时显示
```

### 3. 批量处理

```python
# 批量处理多个请求
prompts = ["问题 1", "问题 2", "问题 3"]
results = llm.batch(prompts)

# 比单独调用快 3-5 倍
```

## 一个现实判断

优化的顺序通常应该是：

1. 先看架构是否过度复杂
2. 再看模型和工具调用是否匹配任务
3. 最后再做缓存、并行和连接池等工程优化

### 3. 响应截断

```python
def truncate_response(response: str, max_tokens: int = 500) -> str:
    tokens = response.split()
    if len(tokens) > max_tokens:
        return ' '.join(tokens[:max_tokens]) + "..."
    return response
```

## 🚀 提升速度

### 1. 并行执行

```python
import asyncio

async def parallel_tool_calls(tools, args_list):
    tasks = [tool.ainvoke(args) for tool, args in zip(tools, args_list)]
    results = await asyncio.gather(*tasks)
    return results

# 3 个工具并行执行，时间从 3 秒降至 1 秒
```

### 2. 懒加载

```python
class LazyAgent:
    def __init__(self):
        self._llm = None
        self._tools = None
    
    @property
    def llm(self):
        if self._llm is None:
            self._llm = load_llm()
        return self._llm
    
    @property
    def tools(self):
        if self._tools is None:
            self._tools = load_tools()
        return self._tools
```
