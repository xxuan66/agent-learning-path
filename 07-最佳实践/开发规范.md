# 开发规范

开发规范的目标不是“写得像教材示例”，而是让 Agent 系统更可维护、更可测试、更容易交接和长期演进。

## 一个现实判断

很多 Agent 项目在早期 demo 阶段看起来很快，但一旦缺少规范，就会很快出现：

- 工具定义混乱
- 提示词和逻辑耦合
- 行为难以复现
- 测试很难写

所以规范并不是锦上添花，而是规模化前的基础条件。

## 📝 代码规范

### 1. 命名规范

```python
# ✅ 好的命名
class CustomerServiceAgent: ...
def calculate_order_total(): ...
MAX_RETRY_ATTEMPTS = 3

# ❌ 坏的命名
class Agent1: ...
def calc(): ...
max = 3
```

### 2. 文档字符串

```python
@tool
def search_knowledge(query: str, k: int = 3) -> str:
    """
    从知识库搜索相关信息
    
    Args:
        query: 搜索查询
        k: 返回结果数量 (默认 3)
    
    Returns:
        搜索结果文本
    
    Raises:
        ValueError: 查询为空时
    """
    if not query:
        raise ValueError("查询不能为空")
    results = knowledge_base.search(query, k=k)
    return format_results(results)
```

### 3. 错误处理

```python
# ✅ 好的错误处理
try:
    result = tool.invoke(args)
except ToolError as e:
    logger.error(f"工具执行失败：{e}")
    return f"抱歉，暂时无法完成该操作：{str(e)}"
except Exception as e:
    logger.exception("未知错误")
    return "遇到未知错误，请稍后重试"

# ❌ 坏的错误处理
try:
    result = tool.invoke(args)
except:
    pass
```

## 一个更实用的建议

如果你做的是 Agent 项目，建议尽量把这些东西分开：

1. Agent 编排逻辑
2. 工具实现
3. 提示词模板
4. 记忆和检索层
5. 配置和密钥

这样后面迁移框架、替换模型、接入新工具时会轻松很多。

## 🧪 测试规范

### 单元测试

```python
import pytest

def test_weather_tool():
    weather = WeatherTool()
    result = weather.invoke({"city": "北京"})
    assert "北京" in result
    assert "°C" in result

@pytest.mark.asyncio
async def test_async_tool():
    result = await async_tool.ainvoke({"param": "value"})
    assert result is not None
```

### 集成测试

```python
def test_agent_workflow():
    agent = create_test_agent()
    response = agent.invoke({"messages": [{"role": "user", "content": "测试"}]})
    assert response is not None
    assert len(response["messages"]) > 0
```

## 🔄 版本控制

### Git 提交规范

```bash
# 格式：<type>(<scope>): <subject>

# 示例
feat(agent): 添加客服 Agent
fix(memory): 修复记忆检索 bug
docs(readme): 更新安装说明
test(tools): 添加工具测试
refactor(core): 重构核心逻辑
```

- [ ] 日志是否充分

- [ ] 性能是否可接受

## 🚀 CI/CD

```yaml
# .github/workflows/ci.yml
name: CI

on: [push, pull_request]

jobs:
  test:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v3
      - name: Set up Python
        uses: actions/setup-python@v4
        with:
          python-version: '3.11'
      - name: Install dependencies
        run: pip install -r requirements.txt
      - name: Run tests
        run: pytest
      - name: Lint
        run: flake8 src/
```
