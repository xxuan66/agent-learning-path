## 关键原则

### 1. 防御性编程

假设一切都会出错，做好错误处理。

### 2. 可观测性

日志、指标、追踪，三者缺一不可。

### 3. 渐进式上线

先小范围测试，再逐步扩大。

### 4. 成本意识

监控 Token 使用，控制成本。
